# 服务器流程
发送一条消息之后,服务器首先判断选中的设备
    选中:
        在线:
            尝试10次发送消息:
                成功:(服务器需要**记录响应**)
                    message class:（实例化）
                        消息ID;
                        消息收到时间;
                        消息状态: 收到 显示 确认;
                    收到消息创建一个实例:
                        状态为收到:写ACK队列**ackrecv**
                    屏幕亮着，LVGL渲染卡片:
                        （其实我觉得那个可以改主页为插槽，一个实例一个卡片，自动排列，紧急最先，如果有同类型，时间最早的靠前）
                        状态改变为显示:写ACK队列**ackshow**
                    摇一摇:
                        状态改为确认:写ACK队列**ackshake**
                    备注:所有实例不直接响应,而是写spiffs,如果遇到写入失败,尝试3次（有那个宏可以用）,如果还是失败,使用弹窗提醒,状态不能跳（收到直接to确认）
                    实例持久化:对于未确认的消息nvs持久化处理,直到本实例**ackshake**成功，彻底删除本实例
                失败:
                    记录失败日志,这个设备标记发送失败,只标记这个设备
        不在线:
            存放在缓冲区,等待设备上线,进入在线流程
    未选中:
        不处理


# SPIFFS ACK队列（可采用json）
消息ID | 时间 | ACK类型 | 尝试次数

ACK类型
-ackshow: 已经显示
-ackrecv: 已经收到
-ackshake: 已经确认

# 响应机
把spiffs里面的响应按照时间顺序依次响应（old优先）尝试10次失败弹窗警告，删除本条ack

# 备注
所常量可config（尝试次数等）
