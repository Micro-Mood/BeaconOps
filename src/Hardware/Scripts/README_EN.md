# ESP Flashing Scripts

A small set of generic ESP-IDF flashing scripts, not specifically tied to BeaconOps. Drop the whole `Scripts/` folder into any project root and it just works — no edits required. Mainly here so I don't have to retype esptool arguments every time I debug.

## Directory Structure

```
<project-root>/
├── main.bin        <- merged firmware binary generated by WSL build.sh
└── Scripts/
    ├── flash.bat   <- flash firmware
    ├── fs.bat      <- pack and flash SPIFFS
    ├── spiffs.py   <- SPIFFS image generator (ESP-IDF spiffsgen-based)
    ├── fs.bin      <- generated SPIFFS image (created by fs.bat)
    └── README_EN.md
```

The SPIFFS content directory can be any path and is passed in as an argument when flashing.

---

## flash.bat — Flash Firmware

Flashes the `main.bin` file from the parent directory using esptool.

**Usage:**

```bat
flash.bat [CHIP] [FREQ] [PORT]
```

| Argument | Description | Default |
|---|---|---|
| CHIP | Chip model | `esp32c3` |
| FREQ | Flash frequency | `80m` |
| PORT | COM port | `COM3` |

If `main.bin` does not exist, the script exits with an error.

**Examples:**

```bat
:: Use defaults
flash.bat

:: Specify chip and port
flash.bat esp32s3 80m COM5
```

---

## fs.bat — Pack and Flash SPIFFS

This script performs two steps:

1. Pack the specified directory into `fs.bin` using `spiffs.py`
2. Flash `fs.bin` to the SPIFFS partition using esptool

**Usage:**

```bat
fs.bat <SPIFFS_DIR> [CHIP] [FREQ] [OFFSET] [SIZE] [PORT]
```

| Argument | Description | Default |
|---|---|---|
| SPIFFS_DIR | SPIFFS content directory (**required**) | none |
| CHIP | Chip model | `esp32c3` |
| FREQ | Flash frequency | `80m` |
| OFFSET | SPIFFS partition offset | `0x190000` |
| SIZE | SPIFFS partition size | `0x270000` |
| PORT | COM port | `COM3` |

> `OFFSET` and `SIZE` must match the `storage` partition in your `partitions.csv`.

**Examples:**

```bat
:: Only provide the SPIFFS directory
fs.bat ..\FS\SPIFFS

:: Full argument list
fs.bat ..\FS\SPIFFS esp32c3 80m 0x190000 0x270000 COM5
```

If packing fails or the source directory does not exist, the script stops before flashing.

---

## Notes

- `esptool.exe` is bundled inside `Scripts/`; no system PATH setup is required
- `python` must be available; `spiffs.py` uses only the Python standard library
- Put the device into download mode before flashing if auto-reset is unavailable
- `main.bin` is generated by the WSL-side `build.sh <WindowsUsername>` script and copied into the project directory automatically
- Recommended order: flash `fs.bat` first (filesystem), then `flash.bat` (firmware)
